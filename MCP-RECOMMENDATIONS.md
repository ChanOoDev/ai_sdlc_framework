# MCP Recommendations

## Must Have

| MCP | Purpose |
|---|---|
| Filesystem | Read/write project files |
| GitHub | Issues, pull requests, code review |
| Supabase | Auth, schema, RLS, storage |
| PostgreSQL | Query and inspect database |
| Playwright | E2E testing |
| Vercel | Deployment and preview checks |

## Should Have

| MCP | Purpose |
|---|---|
| Notion | PRD, sprint docs, knowledge base |
| Figma | UI references |
| Browser/Search | Documentation lookup |

## Suggested Usage

- Use custom agents for process control.
- Use third-party MCPs for external tool access.
- Avoid connecting unnecessary MCPs during MVP build.
